NYC Department of Records Newsletter Manager

  • Project requirements
  • Comparison of existing open source products.

    1. Colossus

      • Github Repository
        Pros
      • Provide contact list management features (manage multiple contact lists)
      • Allow design of emails in application (WYSYWIG Editor)
      • Allow upload of HTML template in application - Does not support upload from local file, but you can copy source HTML code anf paste in the given template box.
      • Store newsletter statistics - You can see delivery success / failures.
      • Permissions for sending, managing contact list, and editing / creating newsletters - There is only one admin account in this system.
        Cons
      • Allow emails to be sent out - System has this feature but I haven't successfully sent.
      • Ability to provide attachments - Does not support upload attachments.
      • Login with NYC.ID - Not support NYC.ID

    2. Listmonk

      • Github Repository
        Pros
      • Allow emails to be sent out - It works on editor's demo, but I can test it on my local system.
      • Allow design of emails in application (WYSYWIG Editor)
      • Allow upload of HTML template in application - Does not support upload from local file, but you can copy source HTML code anf paste in the given template box.
      • Ability to provide attachments - Only pictures support here and there is an attachment button but it doesn't work)
      • Store newsletter statistics - You can see delivery success / failures.
      • Permissions for sending, managing contact list, and editing / creating newsletters - Only one admin Account
        Cons
      • Login with NYC.ID - Does not support NYC.ID

    3. Mail For Good

      • Github Repository
        Pros
      • Provide contact list management features (manage multiple contact lists)
      • Allow design of emails in application (WYSYWIG Editor)
      • Allow upload of HTML template in application - Only support template email in CSV format.
      • Ability to provide attachments - Only support uploading one attachment.
      • Store newsletter statistics (delivery success / failures) - Visualization is provided.
      • Permissions for sending, managing contact list, and editing / creating newsletters - Only one admin Account to give permissions
        Cons
      • Login with NYC.ID - Does not support NYC.ID
      • Allow emails to be sent out - System has this feature but I haven't successfully sent.
      • Based on Amazon SES, so account is limited by Amazon. - This project uses AWS Simple Email Service to send emails and it costs money to send ($0.10 per 1000 emails) and you have limits on emails. Details on Amazon SES service
