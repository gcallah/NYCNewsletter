Django-Newsletter



links:

documentation: https://django-newsletter.readthedocs.io/en/latest/

github: https://github.com/dokterbob/django-newsletter


Features:

  • Users can be added and removed from a subscription.

  • Html code can be set as the template for a message to be sent. If html code is not used as a template. A premailer is used by this app which allows embedded CSS to be translated into inline CSS. HTML code can be uploaded so an email can be made.

  • Attachments and add ons can be managed with this app. Allows widget integration and attachments.

  • Capable of sending emails to the recipients listed within the system.

  • It has a smart queueing system.



Pros:

  • Code is written in Django python. This is beneficial for the team working on it familiar with django.

  • It has been tested in sending a newsletter to large groups of recipients, with an estimate of at least 15,000 recipients.

  • It has worked in multiple working environments.

  • Tests are run to this day and a team is active in working on the product.



Cons:

  • It doesn't appear to contain any statistic analytical tool to analyze emails sent. This may make it difficult to not know if it was successful or not.

  • It is confirmed to send to a large pool of recipients and the app may not be suitable for larger amounts of emails being sent. However, the app is in development and it is possible that this will improve. The problem is that it may take time for a larger pool to be possible. However it depends on how many the department of records wants to send emails to. It may be an option.

  • It is still in development and certain features are missing from it. Templates may not be as update with the competition.

  • It doesn't appear to possess any statistical tools to keep track of whether emails were read or not. It maybe harder for the NYC department of records to rely on for certain events.
